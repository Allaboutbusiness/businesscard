# ownerskr.com 자동발행 연동 (다른 PC Claude Desktop / blog_automation)

기존 자동발행 로직은 그대로 두고, **발행 마지막에 "우리 사이트 업로더" 한 줄만 추가**합니다.

## 준비
1. 그 PC 환경변수 `OWNERSKR_PUBLISH_TOKEN` = Vercel의 `PUBLISH_TOKEN` 값.
2. `pip install requests`.
3. `ownerskr_uploader.py` 를 기존 자동발행 스크립트 옆(예: `uploaders/`)에 둔다.

## 사용 (기존 흐름 끝에 추가)
```python
from ownerskr_uploader import publish_to_ownerskr

# 글 생성·기존 발행(네이버 등) 직후:
url = publish_to_ownerskr(
    제목_문자열,
    본문_마크다운,            # ~3000자, 사진은 함수가 알아서 본문에 삽입
    image_paths=[대표사진경로, 추가사진1, ...],  # 없으면 생략 가능
    category="post",          # 자동글="post", 공지="notice"
)
print("사이트 발행:", url)     # https://ownerskr.com/post/123
```
끝. 즉시 사이트·사이트맵에 반영됩니다.

## API 계약(직접 호출 시)
- `POST /api/upload` — 헤더 `Authorization: Bearer <TOKEN>`, JSON `{filename, contentType, dataBase64}` → `{url}`
- `POST /api/posts` — 같은 헤더, JSON `{category, title, body(markdown), coverImage?}` → `{id, url}`
- 사진은 upload로 먼저 URL을 받아 body의 `![](url)` 또는 coverImage에 사용.
